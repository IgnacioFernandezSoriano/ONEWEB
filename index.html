Your Regulatory Mission

Four objectives.
One foundation.

Every postal regulator shares the same mandate. Meeting it requires one thing: objective, independent, credible data that every stakeholder accepts without dispute.

01
Quality of Service Control
Verify operator compliance with service standards and take corrective action when they fall short.
02
System Quality Improvement
Support Designated Operators in identifying performance gaps and driving improvements across the system.
03
Consumer Protection
Resolve complaints with credible evidence. Ensure transparent public reporting and nationwide coverage.
04
Licence Management
Evaluate operators objectively for licence renewals and set clear, enforceable operating conditions.
How ONE Addresses the Challenges

Three challenges.
Three structural solutions.

One UN-endorsed platform purpose-built for regulatory independence.

01
Regulators need a real-time view of network performance — not just historical reports — to act proactively.
End-to-End & Network Visibility

Through sampling or real postal events, ONE provides a near real-time view of the entire postal journey — from pick-up to final delivery.

02
Relying on a single data source — often the operator itself — undermines the independence regulators need.
Independent & Verifiable Data

An independent panel of senders with automatic EDI and RFID capture generates data that is unbiased, verifiable, and accepted by all parties.

Without a common framework, comparing operators or benchmarking over time is unreliable — and legally contestable.
A UPU-Endorsed Global Standard

The UPU provides a single, consistent methodology recognised by governments, operators, and consumers — built on 15+ years of measurement experience.

Introducing ONE MS

One Management System.
Built for regulators.

ONE MS — One Management System — is the UPU's dedicated platform for domestic postal quality measurement. Designed exclusively for national postal regulators, it gives authorities the independent, objective, and credible data they need to enforce service standards, protect consumers, and drive systemic improvement.

Unlike generic analytics tools, ONE MS is purpose-built for the regulatory context: it integrates E2E measurement, network diagnostics, and operational oversight into a single, unified system — eliminating the need for multiple disconnected tools and manual data reconciliation.

E2E — End-to-End Measurement

The black box approach:
what matters is the total time

Independent panelists dispatch and receive real postal items. The postal network between origin and destination is not instrumented — the E2E transit time is compared directly against the J+K standard to determine compliance.

Definition
Total transit time between two cities for a specific carrier and product.
Method
Independent panelists or barcode scanning register dispatch and receipt times. Multi-carrier, multi-product: letters, parcels, express, registered mail.
Outcome
Verified compliance with delivery standards. Evidence base for regulatory action, tariff decisions, and licence management.
01
Origin panelist — City A
An independent panelist dispatches a real test item from City A using the operator's normal service. The injection timestamp (T inject) is recorded automatically via the ONE Agent App. The panelist is fully independent — not affiliated with the operator.
→ Panelist Management · Agent App · Allocation Plan
02
The postal network — a measured black box
E2E does not instrument the internal network. The sorting centres, transport legs, and delivery offices between City A and City B are not individually tracked. What matters is the total transit time. If performance fails, Network Diagnostic opens this box to find exactly where.
→ E2E Events DB · Compliance Report · Territory Equity Report
03
Destination panelist — City B
A second independent panelist in City B receives the item and confirms delivery via the app. The delivery timestamp (T delivery) is recorded. The system calculates: T delivery − T inject = E2E transit time. This is compared against the J+K target to determine compliance.
→ J+K Performance Report · Regulatory Compliance Report

Diagnostic — Network Measurement

Inside the black box:
pinpoint where delays occur

Network Diagnostic extends E2E by instrumenting every node in the postal chain. RFID readers at sorting centres and transport checkpoints capture exact timestamps at each stage — turning the black box into a transparent, measurable system.

Definition
Extends E2E by measuring intermediate points — sorting centres, transport legs, and post offices.
Method
RFID readers at facility entry/exit gates capture exact timestamps. Integrated with carrier systems via API. No manual intervention required.
Outcome
Identifies root causes of E2E failures. Enables targeted corrective action. Supports network black spot detection and centre efficiency benchmarking.
01
Origin Sorting Centre
RFID readers at entry and exit gates capture the exact time each item spends inside this facility. The system measures: time from arrival → processing → departure. Compared across all routes and centres to identify operational outliers.
→ Postal Centre Diagnostic · Centre Efficiency Ranking
02
Transport Legs
The time between departure from one centre and arrival at the next. RFID captures exact loading/unloading timestamps. The system builds a reliability profile for every transport link: on-time rate, average delay, variance. Green Lanes and Black Spots are ranked automatically.
→ Transport Reliability Analysis · Network Black Spots
03
Intermediate & Destination Centres
Intermediate centres are often hidden bottlenecks — items may arrive on time but accumulate delays before onward transport. Destination centre delays directly cause consumer-facing failures. Diagnostic isolates each contribution to the total E2E time.
→ Postal Centre Diagnostic · Territory Equity Report

Diagnostic Data Capture

Three technologies.
One unified diagnostic layer.

Network Diagnostic analysis requires data captured at every node of the postal network — sorting centres, transport links, and delivery offices. ONE MS integrates three complementary capture technologies, giving regulators a complete and independent view of internal network performance.

Technology 01

RFID Tracking

Radio Frequency Identification tags are attached to test items at injection. As items pass through each facility — sorting centres, transport hubs, delivery offices — fixed RFID readers capture the timestamp automatically, without any human intervention.

Capture method
Passive UHF RFID tags read by fixed antenna arrays at facility entry/exit points
Data captured
Arrival and departure timestamps per facility, dwell time, processing sequence
Diagnostic value
Pinpoints exactly where time is lost within the network — isolating process gaps from resource gaps at each sorting centre
Reports enabled

Barcode Scanning

Independent panelists use the ONE Agent App — integrated with Telegram — to scan the barcode of each test item at injection and delivery. The AI assistant validates each scan in real time, captures GPS coordinates, and records the precise timestamp, eliminating manual data entry errors.

Capture method
Mobile barcode scan via Telegram bot — no dedicated hardware required
Data captured
Injection timestamp (T inject), delivery timestamp (T delivery), GPS location, panelist ID
Diagnostic value
Provides the E2E transit time baseline — the primary evidence layer for J+K compliance and regulatory reporting
Reports enabled

Technology 03

Designated Operator Database Integration

ONE MS connects directly to the Designated Operator's internal tracking and operational databases via a secure API gateway. This integration enriches the independent measurement data with the operator's own records — enabling cross-validation, anomaly detection, and a complete end-to-end audit trail that neither party can manipulate unilaterally.

Integration method
Secure REST API gateway — read-only access to operational tracking events and facility data
Data captured
Operator scan events, facility processing records, transport manifests, delivery confirmations
Diagnostic value
Cross-validates independent measurements against operator data — identifying discrepancies that indicate process failures or data quality issues
Reports enabled
Centre Efficiency Ranking · Transport Reliability Analysis · Network Black Spots & Green Lanes · Regulatory Compliance Report
